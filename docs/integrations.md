# Integrations Reference

> **Load this doc when:** working with Figma Code Connect mappings, publishing packages,
> or integrating with external design or publishing tooling.

---

## Figma Integration

Components may have Figma Code Connect mappings for design-to-code workflows.

- Check for existing Code Connect mappings before creating new ones
- Follow Figma component naming conventions when creating mappings
- Mapping files use the `.figma.js` extension
- See existing mappings in the codebase for examples

---

## Package Publishing

All packages are published to npm under the `@skyscanner/` scope.

- Follow semantic versioning (patch / minor / major)
- Update changelogs for all releases
- Coordinate with the design team for major API changes
- Packages are published from CI — do not publish manually
